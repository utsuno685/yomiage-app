# きくドキ — 文書読み上げアプリ

PDF・Word・Markdown・テキスト・HTMLの文書を、Gemini TTSの自然な音声で聞けるサーバーレスPWA。
[要件定義書](../要件定義書_文書読み上げアプリ.md) v3 に基づく実装。

## 特徴

- **スマホ単体で完結**：ファイル取り込み・変換・保存・再生をすべてブラウザ内で処理（文書は端末外に出ない）
- **Gemini TTS**（無料枠）で自然な音声。合成済み音声は端末にキャッシュし、聞き直し・続き再生はAPI呼び出しなし（オフライン可）
- **続きから再生**：文単位で位置を記憶し、少し手前から再開
- **読み上げ中の文をハイライト**、タップでそこから再生、目次ジャンプ、速度0.5〜3.0倍
- **ロック画面から操作可能**（MediaSession）
- APIキーがない場合はブラウザ内蔵音声（Web Speech API）で動作

## 使い方

### 1. ローカルで起動（PC）

```
cd yomiage-app
python -m http.server 8340
```

ブラウザで http://localhost:8340 を開く。

### 2. Gemini APIキーの設定

1. [Google AI Studio](https://aistudio.google.com/apikey) でAPIキーを無料発行（カード登録不要）
2. アプリの ⚙設定 → 「Gemini APIキー」に貼り付け

キーは端末内（IndexedDB）にのみ保存されます。
無料枠では送信テキストがGoogleのサービス改善に使われることがあります。機密文書には使わないでください。

### 3. スマホで使う（GitHub Pagesに配置）

1. GitHubリポジトリを作成し、`yomiage-app` の中身をプッシュ
2. リポジトリの Settings → Pages → Branch: main を選択して保存
3. 発行されたURL（`https://<ユーザー名>.github.io/<リポジトリ名>/`）をスマホで開く
4. ブラウザのメニューから「ホーム画面に追加」→ アプリとして起動できる

※ 文書・APIキーは各端末内で完結するため、URLが公開されても作者側にコスト・データ流出のリスクはありません。

## 対応形式

| 形式 | 備考 |
|---|---|
| .pdf | ヘッダー/フッター・ページ番号の除去、2段組みの順序補正、参考文献スキップ（設定） |
| .docx | 見出し構造を保持 |
| .md | 見出し→目次化。コードブロック・表・図は省略アナウンス |
| .txt / .html | 対応 |
| .epub | 未対応（将来拡張） |

## 構成

```
index.html      アプリ本体（3画面：ライブラリ / 再生 / 設定）
js/app.js       UI制御
js/extract.js   ファイル形式ごとのテキスト抽出（pdf.js / mammoth.js はCDNから取得）
js/player.js    再生エンジン（チャンク合成・先読み・キャッシュ・MediaSession）
js/tts.js       Gemini TTSクライアント（429リトライ・WAV変換）
js/db.js        IndexedDB（文書・音声キャッシュ・再生位置・設定・使用実績）
sw.js           Service Worker（オフライン起動）
manifest.json   PWAマニフェスト
```

## 無料枠での運用について

- Flash系TTSモデルはGemini API無料枠で無料（2026年6月時点）。レート制限があるため、合成は直列化し先読みは2チャンクまでに制限している
- 429（制限超過）時は自動で待機・再試行し、続かない場合はブラウザ内蔵音声にフォールバック（設定でオフ可）
- 設定画面で今月の合成時間とキャッシュ使用量を確認できる
